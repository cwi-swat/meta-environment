A Taxonomy of debugger features

First we define some basic notions, that will be used in the features described later.

  • Expressions:
    • Constants
    • Variable values
    • Function calls
    • Complex expressions
  • Variables:
    • Scoped variables
  • Source position format:
    • Line
    • Line/column pair
    • Line/column area
    • Instruction source text
    A multithreaded environment often results in a set of positions, instead of a single position.
  • Observation ports:
    • Continuous
    • On halt
    • Every source line
    • When statement matches regexp
    • Call
    • Exit
    • Redo
    • Fail
    • Unify
    • Exception
  • Observation conditions:
    • Variable change
    • Pass count
    • Expressions

The actual features follow here. They closely depend on the notions defined above.

  • Source tracking:
    • Visualization moment:
      • Observation ports
      • Observation conditions
    • Source position format
  • Breakpoints:
    • Observation ports
    • Observation conditions
  • Watchpoints:
    • Observation ports
    • Observation conditions
    • Visualization objects:
      • Variable values
      • Expressions
  • Execution control:
    • Single step
    • Step over
    • Run
    • Run until parent frame
    • Change current point of execution (at observation port)
  • State changes:
    • Variable assignment (with expressions)
    • Call functions with side effects
  • Stack trace:
    • Source position
    • Local variables
In the future, I also want to create a taxonomy of profiler features.
  • Observation points
  • Count
  • Time
