Process List

The process list is an overview of the currently active processes that are running. The process tree has the following functionality:

  • Grouping process instances
  • Setting/Unsetting process breakpoints
  • Showing/Hiding processes in other views
  • Displaying source code breakpoints

Grouping process instances

When a process has only one instance, it will be displayed as a regular item in the process tree. When a process has multiple instances, the instances will be grouped together into a process group. Multiple instances will all be shown by default but by double clicking on the parent process, all the process instances will be represented as one single group.

Setting/Unsetting process breakpoints:

Process breakpoints can be set on process instances. Clicking on the breakpoint column will either set or unset the process breakpoint. If there are multiple instances of a process and a process instance is set, the process group will also display a breakpoint icon, depicting that one of it's instances has a process breakpoint.

Showing/Hiding processes in other views

The checkbox next to the process depicts wether or not it is visible in other views. Also by selecting a process in this view, the same process will be highlighted in other views. For example the selected process in the Process List will be drawn green in the MSC and used in the Process Details view as the selected process.

Displaying source code breakpoints

The process tree will display wether or not a process has a source code breakpoint. When there is only one instance of a process, this icon will appear next to it. When there are several instances of the process, the icon is placed next to the process group.
