The Asf+Sdf Meta-Environment

This page describes the second generation Asf+Sdf Meta-Environment developed at CWI and UvA.

The purpose of the Asf+Sdf Meta-Environment is to support the development of language definitions in Asf+Sdf. Asf+Sdf is an algebraic specification formalism baised towards describing the syntax and semantics of (programming) language. The main application areas of Asf+Sdf are language prototyping, and reengineering.

How to use the Asf+Sdf Meta-Environment

The Asf+Sdf Meta-Environment is still in a pre-release state. Both at CWI and at the UvA a pre-release version of the new Asf+Sdf Meta-Environment is installed:

  • At the UvA: /home/gipe/petr/bin (meta: 1 Dec 1999, 09:57)
  • At CWI:
    • /ufs/gipe/petr/linux/i686/bin (meta: 24 Sep 1999, 16:47)
    • /ufs/gipe/petr/irix/mips/bin (meta: 13 Oct 1999, 17:29)
    • /ufs/gipe/petr/sunos/sparc/bin (meta: 5 Nov 1999, 09:08)

The date and time between the brackets are the installation time of the executables.

By adding the appropriate to our `paths' the new Asf+Sdf Meta-Environment can be activated via the command `meta'.

It is possible to create a file `meta.conf' which contains all search paths that have to be inspected in order to obtain all modules. If all modules are in the same directory please create this file as well, but with only `.' as content. If you are planning to use the compiler then make sure that the variable `COMPILER_OUTPUT' is initialized if you want the generated C files and Makefiles in a separate directory, e.g.,

  • setenv COMPILER_OUTPUT /home/user/c-source
This mechanism may changed in the near future.

Known bugs and other shortcomings

The current implementation of the new Asf+Sdf Meta-Environment is still in a pre-release phase. One of the major changes is the redesign of the ToolBus script. The following list is far from complete but if you run into a problem which is not on the list please contact us.

  • The current implementation of the structure editor is extremely slow.
  • If more than one module can not be found one of the pop up menu of the user interface gets into trouble.
  • The manipulation of `empty' modules is not full proof.
  • The parsing of the text of the focus if the focus is not the entire `program' is not yet possible.
  • The propagation of modifications in the syntax section of a module to the rest of the specification is not yet fully tested.
