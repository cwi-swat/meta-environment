6.1 SDF specific well-formedness checks

Undeclared sorts

If a sort is used in some lexical or context-free syntax rule and it is not defined in some visible sorts section this will be reported. This is in fact no error, but may be caused due to a typo.

Double declared symbols

If a symbol occur in more than one visible sorts section this will be reported. This is again not an error.

Defined symbols without corresponding productions

If a sort is defined but no corresponding production with this symbol as right hand side is can be found. Such a symbol can not be used as a start symbol by the parser, because it produces in fact empty.

Redefinition of aliased symbols

This is a real error, because a redefinition of an aliased symbol, means that this symbol can be replaced in two different ways.

Illegal attributes

Certain attributes are only useful in combination with production rules of a specific form.
M.G.J. van den Brand and P. Klint, September 18, 2003
