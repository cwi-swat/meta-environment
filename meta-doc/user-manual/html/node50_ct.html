Attributes of Lexical and Context-free Functions

The definition of a lexical or context-free functions may be followed by attributes that define additional (syntactic or semantic) properties of that function. The attributes are written between curly brackets after the non-terminal in the right hand side. If a production rule has more than one attribute they are separated by commas. 

context-free syntax
   "{" {Attribute ","}* "}" -> Attributes {cons("attrs")}
                            -> Attributes {cons("no-attrs")}
The following syntax-related attributes exist:

  • bracket allows the definitions of parenthesis and other kinds of brackets that are mostly used for overruling the priorities of operators in expressions (see Bracket Functions[Section ]BracketFunctions).

  • left, right, non-assoc, and assoc are used to define the associativity of functions (see Priorities[Section ]Priorities).

  • prefer is used to indicate that the attributed function should always be preferred over other functions (without this attribute) in certain cases of syntactic ambiguity (see Preferring, Avoiding or Rejecting Parses[Section ]PreferAvoidReject).

  • avoid is used to indicate that a function should only be used as a last resort in certain cases of syntactic ambiguity (see Preferring, Avoiding or Rejecting Parses[Section ]PreferAvoidReject).

  • reject can be used to explicitly forbid certain syntactic constructs (see Preferring, Avoiding or Rejecting Parses[Section ]PreferAvoidReject).

The remaining attributes define semantic properties of a function:

  • constructor declares a function to be a constructor function, this means that for this function no equations may be defined with this function as outermost function symbol in the left hand side.

  • memo declares a function to be a memo function for which all calls and results will be cached during evaluation (see Memo Functions[Section ]MemoFunctions).

  • traversal is used to declare so-called traversal functions that greatly simply the specification of functions that have to visit (parts of) a term (see Traversal Functions[Section ]Traversal).

  • arbitrary ATerms may also be used as attributes. In the context-free syntax definition of the Attributes, the ATerms cons("attrs") and cons("no-attrs") are used. The cons attribute is used by other tools, such as ApiGen.

Not all combinations of attributes make sense. If one uses the attribute left in combination with bracket, right, assoc or non-assoc, this will result in an error message. The combination of avoid and prefer does not make sense either. Furthermore, the combination of the traversal attributes is also very strict.
