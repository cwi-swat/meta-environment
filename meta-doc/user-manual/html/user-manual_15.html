2.6 Sorts

Sorts are declared by listing their name in a sorts section of the form: 
sorts
  <Symbol>*

The keyword sorts is in fact misleading, since not only plain Sorts can be declared, but more complex Symbols can be declare in the sorts section as well.

The definition of symbols in the sorts section is two-fold. First of all, it is required that all sorts that occur in some symbol in the specification are declared. Secondly, symbols declared in the sorts section act as start symbol for the underlying grammar. This has the effect that input sentences corresponding to these symbols can be parsed.

Sorts sections can either be hidden or exported. The effect of defined symbols as start symbols for the grammar may lead to an explosion of start states for the parser and thus lead to performance loss. To prevent this it can be advisable to hide sort sections. The symbols defined are only visible in the module containing this hidden sorts section, but not in modules importing this module.

Recall that a sort name should start with a capital letter and may be followed by letters and/or digits. Hyphens (`-') may be embedded in sort names. There is one predefined sort name (LAYOUT). It is described in Lexical Syntax.
