When to use the Asf+Sdf Meta-Environment?

The Asf+Sdf Meta-Environment is an interactive development environment (IDE) for meta programming. It can be used to generate implementations of

  • compilers,
  • interpreters,
  • source-to-source transformers,
  • source code analyzers,
  • pretty printers (formatters),
  • type-checkers,
  • syntax highlighters,
  • source code generators,
  • and even full blown IDE's.

The generative process is controlled by a syntax definition of a (programming) language, and a semantical description of the manipulations you want to express on the language.

The Asf+Sdf Meta-Environment can help you if

  • You have to write a formal specification for some problem and you need interactive support to do this.
  • You have developed your own (application) language and want to create an interactive environment for it.
  • You have programs in some existing programming language and you want to analyze or transform them.

The Asf+Sdf Meta-Environment contains a number of formalisms, or domain specific languages, for the definition of several aspects of a programming language. For the syntactic aspect, we have Sdf. For semantical analysis and transformation, we have Asf. The combination of the two is called Asf+Sdf. For pretty-printing, we use the BOXlanguage. Finally, for fact analysis and elaboration we use RSCRIPT. See Figure 61 for an overview.

The four combined formalisms allows the definition of syntactic as well as semantic aspects of a (programming) language. It can be used for the definition of languages. In addition it can be used for the formal specification of a wide variety of problems. We provide you with:

  • A general-purpose algebraic specification formalism based on equational logic (Asf+Sdf)
  • Modular structuring of specifications.
  • Integrated definition of lexical, context-free, and abstract syntax.
  • User-defined syntax, allowing you to write specifications using your own notation (Sdf)
  • Complete integration of the definition of syntax and semantics (Asf+Sdf)
  • High-level definition of pretty printers (Asfand BOX)
  • High-level analysis and fact manipulation. RSCRIPTis a language for the manipulation of facts encoded as relations. The RSCRIPTlanguage is relatively new, and is not fully integrated into the Asf+Sdf Meta-Environmentyet.

The Asf+Sdf Meta-Environment IDE offers:

  • Syntax-directed editing of Asf+Sdf specifications.
  • Automatic syntax highlighting.
  • Testing and interactive debugging of specifications.
  • Compilation of Asf+Sdf specifications into dedicated commandline tools.
  • A collection of ready-to-use components that can be composed into your own IDE for any programming language.

The advantages of creating (interactive) environments in this way are twofold:

  • Increased uniformity. Similar tools for different languages often suffer from a lack of uniformity. Generating tools from language definitions results in a large increase in uniformity, with corresponding benefits for the user.

  • Reduced implementation effort. Preparing a language definition requires significantly less effort than developing an environment from scratch. The reason is that the domain specific languages in the Asf+Sdf Meta-Environmentoffer a higher level of abstraction and automation for common language design issues.
