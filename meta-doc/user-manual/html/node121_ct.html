Rewriting a Term using the Evaluator

In order to rewrite a term term.txt using the Asf+Sdf evaluator asfe two inputs are required:

  • The parse tree of term.txt. See parsing of terms[Section ]SEC:parsingofterms how to do this. The result is term.tree.

  • The equations to be used for rewriting. See obtaing of equations[ SectionRef]SEC:obtainingequations for obtaining the equations given an Asf+Sdf specification.

Given this two inputs it is possible to call the interpreter and reduce terms via the command line. This can be performed in two different manners: The first is: 

   asfe -e M.eqs < term.tree > reduct.tree
The second is: 
   asfe -e M.eqs -i term.tree -o reduct.tree

Unparsing[Section ]SEC:unparsing explains how reduct.tree can be converted to a textual representation.
