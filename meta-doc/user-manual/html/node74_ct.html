Variables

Variables are declared in the `variables' section of a module. Like all other entities in a module--except equations--variables may be exported (see Modules[Section ]modules). A variables section consists of a list of variable names followed by a symbol. In fact, a variable declaration can define an infinite collection of variables by using a naming scheme instead of a simple variable name. A naming scheme is a regular expression like the ones allowed in the lexical syntax[ (Section )]lexical-syntax except that sorts are not allowed. A variable may represent any symbol.

In the specification below[in Figure ]CODE:variables, `Id', `Type3', and `Id-list' are examples of variables declared by the naming schemes in the variables section. Strings that occur in the left-hand side of variable declarations should always be quoted.

Figure 39: Variable declarations using naming schemes

Declared variables can only be used when defining equations. It is not allowed to use them in terms.

Ambiguities due to variables are resolved by the Prefer Variables strategy that was discussed in Lexical Ambiguities[Section ]lex-ambiguity.
